Happy Dreamers API

Conecta servicios externos (como Yose) para registrar y consultar eventos de sueño, estadísticas y notificaciones — autenticado con API Keys.
Introducción

Qué es esta API

La API pública de Happy Dreamers expone una superficie REST estable bajo /api/v1, autenticada por API Key. Está pensada para que aplicaciones externas — por ejemplo Yose (asistente de WhatsApp/Telegram/iOS) — puedan, en nombre de un usuario, registrar, editar y eliminar eventos, leer estadísticas y gestionar notificaciones.

  • Base URL: https://<tu-dominio>/api/v1
  • Formato: JSON. Respuestas de éxito: { "data": ... }. Errores: { "error": { "code", "message" } }.
  • Contrato OpenAPI: GET /api/v1/openapi.json (también en docs/api/openapi.yaml).
Autenticación

API Keys

Cada llamada debe incluir la cabecera Authorization con el secreto de la API key:

Authorization: Bearer hd_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxx

Cómo se crea una key

El usuario entra a Configuración → API / Desarrolladores en Happy Dreamers, elige un nombre, los scopes y opcionalmente limita la key a ciertos niños. El secreto se muestra una sola vez; en el servidor solo se guarda su hash sha256.

Seguridad: trata el secreto como una contraseña. No lo incrustes en apps cliente ni en el control de versiones. Si se filtra, revócalo desde la misma pantalla (deja de funcionar al instante).
Permisos

Scopes

Una key solo puede hacer lo que sus scopes permiten; además, si tiene una lista de niños, solo puede operar sobre esos.

ScopePermite
children:readListar y leer niños
events:readLeer eventos
events:writeCrear, editar y eliminar eventos
stats:readLeer estadísticas de sueño
notifications:readLeer notificaciones
notifications:writeCrear/programar notificaciones
Rate limit: 120 solicitudes/minuto por key (con control de ráfaga). Si lo superas recibes 429.
Endpoints

Referencia

GET/me

Identidad de la key: usuario y niños accesibles. Útil para descubrir los childId tras enlazar.

curl -H "Authorization: Bearer $HD_KEY" \
  https://tu-dominio/api/v1/me
GET/children  children:read

Lista de niños accesibles por la key.

GET/children/{id}/sleep-state  children:read

Estado de sueño actual: awake, sleeping, napping o night_waking.

GET/events?childId={id}  events:read

Eventos del niño ordenados cronológicamente.

POST/events  events:write

Registra un evento. Campos por tipo (sleep, nap, wake, night_waking, feeding, medication, extra_activities, note).

curl -X POST https://tu-dominio/api/v1/events \
  -H "Authorization: Bearer $HD_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "childId": "665f0c2b9a...",
    "eventType": "sleep",
    "startTime": "2026-06-01T20:30:00-06:00",
    "endTime": "2026-06-02T07:00:00-06:00",
    "sleepDelay": 15,
    "emotionalState": "tranquilo"
  }'

Ejemplo de alimentación nocturna (usa el flag, no un tipo aparte):

{
  "childId": "665f0c2b9a...",
  "eventType": "feeding",
  "feedingType": "bottle",
  "babyState": "asleep",
  "isNightFeeding": true,
  "feedingAmount": 120,
  "startTime": "2026-06-02T02:10:00-06:00"
}
PATCH/events/{id}  events:write

Actualiza un evento. Parcial por defecto (ej. agregar endTime); si incluyes eventType es un reemplazo completo. Requiere childId en el cuerpo.

curl -X PATCH https://tu-dominio/api/v1/events/EVENT_ID \
  -H "Authorization: Bearer $HD_KEY" -H "Content-Type: application/json" \
  -d '{ "childId": "665f...", "endTime": "2026-06-02T07:15:00-06:00" }'
DELETE/events/{id}?childId={id}  events:write

Elimina un evento.

GET/stats?childId={id}&from={ISO}  stats:read

Métricas de sueño (duración promedio, hora de dormir, despertares, etc.).

GET/notifications?childId={id}  notifications:read
POST/notifications  notifications:write

Lee el historial o crea/programa una notificación (childId, title, message, opcional scheduledFor).

Integración con Yose

Cómo lo usa Yose

Yose guarda la API key del usuario cifrada y la usa como Bearer en cada llamada. El flujo natural-language → acción:

  1. El usuario conecta su cuenta pegando la key (o vía link) → Yose la cifra y guarda.
  2. Yose llama GET /api/v1/me para descubrir los niños y sus childId.
  3. Un mensaje como "Mateo se durmió a las 8:30" se mapea a POST /api/v1/events con eventType: "sleep".
  4. "¿Cómo durmió anoche?" se mapea a GET /api/v1/stats.
El detalle de implementación del lado de Yose (servicio + intents) corresponde a la Fase 2 del proyecto.
